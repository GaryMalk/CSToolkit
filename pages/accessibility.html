Text to speech and speech to text

Speech to text and text to speech software are useful tools for making programming more accessible. Speech to text software takes vocal input from the user and turns it into text on the computer. Text to speech does the opposite, taking text from the computer and outputting it as audio. Speech to text can help a student who is not able to type for whatever reason. Text to speech can help students who are not able to read text. Both kinds of software are widely available, some IDE's have TTS and STT plugins available, VSCode definitely does. If you are using a web based IDE most browsers have TTS extensions available.

Resources

Text to Speech

  • ttsmaker: Free text to speech online tool.
  • Read Aloud : TTS plugin for firefox, they also have a version for chrome.

Speech to Text

  • Windows 11 speech recognition: Built in STT functionality in Windows 11
  • Apple dictation: Built in STT functionality in Apple products

Hardware

For students that have some physical disability that makes using a computer difficult there may be a hardware solution that can help them. Here are some examples:

  • Braille keyboards: Keyboards that have Braille keys which allow someone who understands Braille to read the keys by touch. There are also Braille displays which can be used to read text from the computer..
  • Eye tracking: With the right software and hardware you can use your eyes to control a computer
  • Adaptive Keyboards: Many special keyboards are made for people who have issues using a standard one. For example, a larger keyboard with depressed buttons for people who have muscle control issues.
  • Foot switches: Allow a person to use their feet to control a computer.

This is just a small sample of the accessibility tools available. This article contains a nice summary of the kinds of tools that are out there.
