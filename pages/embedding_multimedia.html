The Internet today is all about streaming and embedding multimedia, and your website will be more compelling if you take advantage of it. You have learned how to add images, colors, fonts, and styling to your website. This lesson is about adding videos, PowerPoint presentations, and Google Slides with the proper aspect ratio and to have them size responsively with the size of the browser window.

Local Video

For your website's own video, you don't need an iframe. You could simply link it with the <video> tag:
<video class="center" width="80%" controls autoplay>
   <source src="images/video/myvideo.mp4" type="video/mp4">
   Your browser does not support the video tag.
</video>

You don't need to set the height or do anything special, because the video element will communicate to the browser what size it needs to stay proportional to the width you select.

To save bandwidth and get better playback performance, however, I do recommend that you upload your videos to YouTube or some other streaming platform instead, and insert them into your site with the iframe element. You may do it either way. For short animations, this might not matter. But if we stream from other platforms, we need the <iframe> tag.

Using the <iframe> tag

The purpose of iframe is frame another webpage within your site. This is commonly done for framing videos. The basic attributes are src, width, and height:

<iframe src="/default.asp" width="100%" height="300">
</iframe>

For a full list of attributes, see: https://www.w3schools.com/tags/tag_iframe.ASP

For security, iframe is not able to communicate to the browser or control your webpage. This unfortunately means that it can't even communicate its size. You will notice if you grab the default embed code from YouTube, it will give you preset width="560" and height="315". You can change that, but you want to keep the right aspect ratio or your video height will be chopped off (if too short) or have a black section (if too long). Width as a percentage is based on the screen size, so iff you set the width to 100% you will get your video to be the full width of your screen. Perfect, you think, just set them both to 100%. Wrong. Because height doesn't know what 100% means. It will just give you 150 pixels which will suck.

What you need to do is to put the iframe inside another container and let the container handle the height through a little trick. What you need for YouTube as well as PowerPoint and Google Slides is a 16:9 aspect ratio, which is precisely 56.25%. We know the width, it's just the size of your screen or of the container you put the iframe in (more about that in a moment). What you need then, is a height which is precisely 56.25% of the width. The trick to set this is to use padding-bottom (or you could use padding-top instead). Set padding-bottom: 56.25% on the container.

But you also have to get the iframe to play along. To do that, you have to use some of the tricks we learned about with positioning. What we want is for the iframe to exactly match the container. To do that, we will use absolute positioning and set top and left to 0. And as you learned, absolute positioning bases the position of the element on its first ancestor that uses relative positioning, so we have to set the parent container to relative. We will also set overflow: hidden to prevent content from overflowing and causing unwanted scrollbars or layout issues.

YouTube and PowerPoint will also give some values you don't need. You want to delete the height and width attributes from the html code and let the CSS styling handle all of that, as we've discussed. But they also give you frameborder="0" to prevent black lines around your media. But the html validator tool rejects that as outdated. You want to delete that from the html, too, and put border: none on the iframe in CSS instead.

Putting it all together, your CSS looks like this:

.video-container {
  position: relative; /* Required for absolute positioning of iframe */
  width: 100%;
  height: 0; /* Creates an element with no height */
  overflow: hidden; /* Prevents overflowing content creating scrollbars */
  padding-bottom: 56.25%; /* 9 / 16 = 0.5625 */
}

.video-container iframe {
  position: absolute; /* Positions iframe relative to the container */
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  border: none;
}
and your HTML code looks like this:
<div class="video-container">
  <iframe
    src="https://www.youtube.com/embed/VIDEO_ID"
    title="YouTube video player"
    allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
    allowfullscreen
  ></iframe>
</div>

NOTE: I will deduct points if you don't change the title from the default "YouTube video player" to whatever your video or presentation is actually about!

Your video container doesn't need to be sized with width=100% of your screen area, though, only your iframe needs to do that. You can adjust the width to fit your layout. You can even have multiple widths by adding extra classes. For example, if I wanted some videos to be full page (100%), some to be half page (say 45% to allow some padding), and some quarter page (again, I'll use 20% to allow myself some wiggle room), I can do that by adding customized classes to override the width. In the CSS, I stack classes with the dot "." In the HTML, I stack them by having spaces between them:

.video-container {
  position: relative; /* Required for absolute positioning of iframe */
  width: 100%;
  height: 0; /* Creates an element with no height */
  overflow: hidden; /* Prevents overflowing content creating scrollbars */
  padding-bottom: 56.25%; /* 9 / 16 = 0.5625 */
}

.video-container.half {
  width: 45%; /* will inherit all the above values but override width */
}

.video-container.quarter {
  width: 20%; /* will inherit all the above values but override width */ }

.video-container iframe {
  position: absolute; /* Positions iframe relative to the container */
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  border: none;
}
and your HTML code looks like this, for videos you want in a quarter width:
<div class="video-container quarter">
  <iframe
    src="https://www.youtube.com/embed/VIDEO_ID"
    title="YouTube video player"
    allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
    allowfullscreen
  ></iframe>
</div>

You only need to adjust the container; leave the iframe formatting exactly as before.

This "padding hack" method will work with all browsers. However, there is also a newer, simpler method that will work with the latest browsers by using the modern aspect-ratio CSS property:

.video-container {
  width: 100%;
  aspect-ratio: 16 / 9; /* Sets a 16:9 aspect ratio */
}
.video-container iframe {
  width: 100%;
  height: 100%;
  border: none;
}
This is much easier, but older browers won't support it. The "padding-hack" is a long-standing method that is safer.

Getting Embed Code

YouTube

  1. Navigate to the video you want.
  2. Click the Share button
  3. Click the Embed button
  4. Copy the code

The code will look something like this:

<iframe width="560" height="315" src="https://www.youtube.com/embed/XlUjyAk_ftA?si=6r6Klf3xzSnRoC6Q" title="YouTube video player" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" referrerpolicy="strict-origin-when-cross-origin" allowfullscreen></iframe>

Be sure to remove the code in red: height, width, and frameborder, and to change the title.

<iframe src="https://www.youtube.com/embed/XlUjyAk_ftA?si=6r6Klf3xzSnRoC6Q"
   title="Steve Martin, Martin Short, & Selena Gomez Q&A"
   allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share"
   referrerpolicy="strict-origin-when-cross-origin" allowfullscreen>
</iframe>

NOTE: You will not be able to view YouTube videos embedded in HTML files that are on your local system. You must upload them to the web server first. When you are working on your page, you will see this Error 153 message. Don't panic - this is normal. You won't be able to test your web page until after you push it to the server. PowerPoint and Google Docs don't have this problem.

PowerPoint

  1. To share a PowerPoint presentation, save it first to a personal OneDrive. Embedding a presentation you've saved the school OneDrive won't work because that's SharePoint. It won't let you share it outside of LWTech. You can tell the difference from the URL: the URL to the LWTech OneDrive is https://lwtech.sharepoint.com/. The URL to a personal OneDrive will start https://onedrive.live.com/.
  2. Open up your OneDrive in a browser. Click My Files and navigate to the saved presentation file.
  3. Click the three dots next to the name to bring up the context menu and select Embed.
  4. When the Embed dialog appears, click "Copy embed code"

The code will look something like this:

<iframe src="https://1drv.ms/p/c/cc1b7ab97fbc565f/IQSmeu-Am8iSR4GYhp0jzuslAXVJjehnbLDe0KKBoYd8pEE" width="402" height="327" frameborder="0" scrolling="no"></iframe>

Again, delete the width, height, and frameborder. There is no title generated by PowerPoint but you should give it one.

<iframe title="Algorithms: Trees"
   src="https://1drv.ms/p/c/cc1b7ab97fbc565f/IQSmeu-Am8iSR4GYhp0jzuslAXVJjehnbLDe0KKBoYd8pEE"
   scrolling="no">
</iframe>

Google Slides

Google Slides doesn't generate embed code for you, but it's simple to create.

  1. Open Google Drive and navigate to your Google Slides presentation. It should be listed on a row.
  2. Click the three dots at the end of the row and select Share.
  3. Change General access to "Anyone with the link" (not "Restricted") and be sure "Viewer" is selected in the drop down, not "Editor".
  4. Copy the link.

The link will look something like this:

https://docs.google.com/presentation/d/1HEQ9mt0JKsIEZ33gcGoc-DS7l73YMIPa3l6nU6BZ7WE/edit?usp=sharing

You want to replace "edit" with "embed". In this case, I also set three values:

  • start=true so the slideshow will begin when the page is loaded
  • loop=true so the show will continuously loop until stopped
  • delayms=3000 set the delay to 3000 microseconds, which is 3 seconds, so the page has a chance to finish loading all the other elements before the slide show begins

You can modify these values, as desired.

The edited URL:

https://docs.google.com/presentation/d/1HEQ9mt0JKsIEZ33gcGoc-DS7l73YMIPa3l6nU6BZ7WE/embed?start=true&loop=true&delayms=3000"

I then added a title and the keyword to allow full screen:

<iframe title="Gary Malkasian: A little about me"
   src="https://docs.google.com/presentation/d/1HEQ9mt0JKsIEZ33gcGoc-DS7l73YMIPa3l6nU6BZ7WE/embed?start=true&loop=true&delayms=3000"
   allowfullscreen>
</iframe>

This iframe will work with the 16:9 aspect ratio box just like PowerPoint presentations and YouTube videos.
